# MiMo Stable

修好了。

MiMo 模型在 Agent 场景下死循环？三刀，切完就走。压测两轮，零循环。

不是"试试看"。是修好了。

## 问题

MiMo（xiaomimimo 全家桶）在 Agent 工具调用时会陷入 Degenerate Loop——同一段话逐字重复 8 次，6 分钟原地踏步。发消息打断它？没用。下次照犯。

**根因：** `reasoning=True` 的模型在长输出时，英文推理 token 的概率分布直接塌缩了。注意力机制聚焦刚输出的 token，自己给自己注射多巴胺。不是 prompt 的问题。是模型底层的 Bug。绝了。

## 修复：三刀

不是调参数。不是试试看。是从物理上切断循环的每一条通路。

### 第一刀：设超时

```json
{"timeoutSeconds": 180}
```

之前：∞ 超时。死循环了？跑去吧，跑 6 分钟没人管。  
现在：180 秒。超时就挂。循环？你自便，我先走了。

### 第二刀：砍输出长度

```json
{"maxTokens": 8000}
```

之前：32000。给循环留了 4 倍放大空间，够它自我复制 8 遍。  
现在：8000。空间不够，循环形成不了。想写长文？分多轮写。工程不是请客吃饭。

关键：只砍 reasoning 模型。`mimo-v2-omni`（reasoning=False）没有这个病的基因，保持 32000。

### 第三刀：行为检测

```
- 连续 3+ 次输出一模一样 -> 当场按住
- 同一个 tool 连续 3+ 次同参数 -> 直接打断
- 打断后重新评估，换路走
```

前两刀是工程。这一刀是兜底。万一前两刀开了小差，第三刀 3 次重复内必抓。

## 完整配置

```json
{
  "models": {
    "providers": {
      "xiaomimimo": {
        "timeoutSeconds": 180,
        "models": [
          { "id": "mimo-v2.5-pro", "reasoning": true, "maxTokens": 8000 },
          { "id": "mimo-v2.5", "reasoning": true, "maxTokens": 8000 },
          { "id": "mimo-v2-pro", "reasoning": true, "maxTokens": 8000 },
          { "id": "mimo-v2-omni", "reasoning": false, "maxTokens": 32000 }
        ]
      }
    }
  }
}
```

拷过去，跑起来。完事。

## 压测

| 项目 | 结果 |
|------|------|

| 语法检查 | 全过 |
| 执行验证 | 全正确 |

| 循环 | 零 |

## 三刀切断了什么

| 刀 | 切断 |
|---|------|
| timeoutSeconds: 180 | 无限循环的时间通路 |
| maxTokens: 8000 | 自我复制的空间通路 |
| 行为检测 | 无感知的逻辑通路 |

## 坟场

| 尝试 | 为什么死 |
|------|---------|
| 调 temperature | 随机性更高->更多循环 |
| frequency_penalty | 1.0 也打断不了 |
| 只检测不改配置 | 看到循环了，然并卵 |
| 压缩上下文 | Bug 在生成层 |
| 强制中文 System Prompt | 降低频率没根治 |
